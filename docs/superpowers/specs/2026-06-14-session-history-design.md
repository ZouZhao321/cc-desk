# CC Desk 会话历史功能 — 需求三棱镜

填写人：ZouZhao
日期：2026-06-14

---

## 【问题层：我要解决什么问题】

**现象：** Claude Code CLI 在 `~/.claude/projects` 下按项目存储对话日志（JSONL 格式）。目前有 39 个项目、366 个会话，总计约 113 MB。这些对话记录了真实的开发过程——调试思路、工具调用、问题解决方案——是很有价值的复盘素材。但目前没有工具能方便地浏览和标注这些会话。

**影响：** 用户（开发者自己）遇到类似问题时，无法快速回溯之前的解决过程，需要重新排查，浪费时间。有价值的经验沉淀在 JSONL 文件中，无法被检索和复用。

**根因（假设）：** Claude Code 的会话日志是机器友好的 JSONL 格式，但缺少人可读的浏览和标注工具。文件分散在多个项目目录下，手动查看不现实。

---

## 【结果层：成功是什么样的】

**成功场景：** 某个开发者在 2026-06-20 遇到了一个 Tauri IPC 超时问题。他打开 CC Desk，切换到"会话历史"页面，看到按项目分组的会话列表。他找到上次在 `cc-desk` 项目中调试 IPC 的会话（通过时间和项目名定位），点进去看到完整的对话记录——当时的排查思路、工具调用、最终解决方案。他给这个会话加了一条备注："Tauri IPC 超时，原因是插件注册顺序"。下次再遇到类似问题，他可以直接通过备注找到这个会话。

**不在本次范围内的事：**
- 跨会话关键词搜索
- 消息级标注（标记特定消息片段）
- 导出/分享功能
- 对话内容编辑
- 标签功能（数据结构预留，UI 不做）

---

## 【验收层：怎么知道做到了】

**验收条件：**

1. 能扫描并展示 `~/.claude/projects` 下所有会话，列表信息完整（时间、项目名、备注摘要、消息数）
2. 点击任意会话能加载并完整显示对话内容（含文本、工具调用、思考过程等所有消息类型）
3. 能为任意会话添加备注，重启应用后备注仍在
4. 有备注的会话在列表页显示备注摘要（前 50 字符）
5. 列表加载时间 < 3 秒（366 个会话量级）

**价值估算（量级即可）：** 现在遇到重复问题平均花 30 分钟重新排查，有了会话历史回溯预计可缩短到 5 分钟以内，每次节省约 25 分钟。按每周遇到 2 次类似问题计算，每月节省约 3.5 小时。

## 功能设计

### 页面结构

在现有 CC Desk 中新增"会话历史"页面，与"配置管理"页面并列。顶部 Tab 切换两个页面。会话历史页面内部用组件状态切换列表视图和详情视图（不用路由）。

### 会话列表视图

- 按项目名分组展示所有会话，每个项目是一个折叠区块
- 每个会话卡片显示：项目名、会话开始时间、备注摘要（如有）、消息数量
- 默认按时间倒序排列
- 点击会话 → 切换到详情视图
- 项目分组支持展开/折叠
- 加载中显示骨架屏

### 对话详情视图

- 顶部信息栏：项目名、会话时间、Claude Code 版本（如有）、消息总数
- 对话内容区：完整展示 JSONL 中所有消息，按时间顺序
  - 每条消息显示角色标签（User / Assistant / System）和时间戳
  - 文本消息：直接显示
  - 工具调用：显示工具名 + 输入参数（代码块格式）
  - 工具结果：显示返回内容
  - 思考过程：折叠显示，点击可展开
  - 附件/系统事件：灰色小字显示
- 左上角返回按钮回到列表

### 标注功能

- 详情页底部备注输入框，自由文本
- 自动保存（失焦或停顿 500ms 触发）
- 存储在 `.cc-desk/annotations.json`，以 session UUID 为键
- 再次进入同一会话时备注自动回填

### UI 设计稿

设计稿位置：`docs/superpowers/specs/2026-06-14-session-history-design.pen`

| 页面 | 说明 |
|------|------|
| Session History - List View | 会话列表页：顶部导航栏（CC-Desk 标题 + Config/Sessions Tab）+ 项目分组卡片列表，每张卡片显示时间、项目路径、消息数、备注状态 |
| Session History - Detail View | 对话详情页：顶部返回栏（Back + 项目信息）+ 对话内容区（用户消息、助手回复、工具调用、思考过程）+ 底部备注输入框 |

### 数据流

- 进入列表页：Rust 扫描 `~/.claude/projects`，提取所有会话元数据，合并备注返回
- 点击会话：Rust 读取对应 JSONL 全量解析，返回消息列表 + 备注
- 保存备注：前端调用 Rust 命令写入 annotations.json（原子写入）
- 加载失败显示错误提示，支持重试
